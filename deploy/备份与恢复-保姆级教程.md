# 备份与恢复 · 保姆级教程

从零到会，照抄命令即可。分四部分：

- **第 0 部分：应急保底备份**（3 分钟，机器要崩时先做这个）
- **第 1 部分：装 rclone 并连上 Google Drive**（一次性，约 10 分钟）
- **第 2 部分：配置每天凌晨 4 点自动备份**（一次性）
- **第 3 部分：新服务器一条命令恢复**

> 约定：项目在 `/home/ubuntu/odds-bot`，用户 `ubuntu`。你的路径不同就把命令里的
> `ubuntu`、`odds-bot` 换成实际值。命令里 `#` 后面是注释，不用输。

---

## ⚠️ 关于本仓库脚本的换行符（重要，先做一次）

脚本在 Windows 上编辑过，行尾可能是 CRLF，Linux 直接跑会报 `bad interpreter`。
`git pull` 之后先净化一次（做一次即可）：

```bash
cd ~/odds-bot
sudo apt-get install -y dos2unix 2>/dev/null || true
dos2unix deploy/*.sh 2>/dev/null || sed -i 's/\r$//' deploy/*.sh
chmod +x deploy/*.sh
```

## 第 0 部分：应急保底备份（机器要崩时，先做这个）

不依赖任何工具，纯 tar + scp，3 分钟拿到一份离线备份。

**① 在服务器打包：**
```bash
cd ~/odds-bot
sqlite3 odds.db "PRAGMA wal_checkpoint(TRUNCATE);"   # 让数据库落盘，拿一致快照
tar czf ~/odds-backup.tar.gz .env odds.db report/ data/
ls -lh ~/odds-backup.tar.gz                          # 确认生成了
```

**② 在你自己的电脑上，把它拉下来**（关键：备份必须离开这台服务器）：

Windows PowerShell 或 Mac/Linux 终端里：
```bash
scp ubuntu@服务器IP:~/odds-backup.tar.gz .
```
> 把 `服务器IP` 换成真实 IP。拉下来后，这份 `.tar.gz` 就是你的保命符，
> 机器崩了也不怕。恢复见第 3 部分。

## 第 1 部分：装 rclone 并连上 Google Drive（一次性）

### 1.1 装 rclone

```bash
sudo -v ; curl https://rclone.org/install.sh | sudo bash
rclone version        # 出现版本号即成功
```

### 1.2 授权（服务器没浏览器，用 headless 方式）

在**服务器**上运行：
```bash
rclone config
```

按提示依次操作（`>` 后是你要输入的）：

| 提示 | 输入 | 说明 |
|---|---|---|
| `e/n/d/...` | `n` | 新建 remote |
| `name>` | `gdrive` | **必须叫 gdrive**，脚本认这个名字 |
| `Storage>` | 找到 `drive` 对应的编号并输入 | Google Drive |
| `client_id>` | 直接回车 | 留空用默认 |
| `client_secret>` | 直接回车 | 留空 |
| `scope>` | `2` | 仅 rclone 建的文件，最安全 |
| `root_folder_id>` | 直接回车 | 跳过 |
| `service_account_file>` | 直接回车 | 跳过 |
| `Edit advanced config?` | `n` | |
| `Use web browser to automatically authenticate?` | **`n`** | ⚠️服务器无浏览器，一定选 n |

选完 `n` 后，rclone 会打印一条命令，形如：
```
rclone authorize "drive" "eyJ...一长串..."
```

**把这条命令复制**，到**你自己有浏览器的电脑**上（本地也先装个 rclone，同样用上面的
install 命令；Windows 去 rclone.org/downloads 下 exe）运行它：

- 浏览器会自动弹出 Google 登录页 → 选你的账号 → 允许授权
- 授权后，你电脑的终端会吐出一段 `{"access_token":...}` 的 token

**把这段 token 整个复制**，粘回服务器 `rclone config` 正在等待的
`config_token>` 处，回车。

剩下：
| 提示 | 输入 |
|---|---|
| `Configure this as a Shared Drive (Team Drive)?` | `n` |
| `Keep this "gdrive" remote?` | `y` |
| 回到主菜单 | `q` 退出 |

### 1.3 验证连通

```bash
rclone lsd gdrive:        # 能列出你 Drive 里的文件夹 = 成功
```

## 第 2 部分：配置每天凌晨 4 点自动备份（一次性）

### 2.1 先手动跑一次，确认全链路通

```bash
bash ~/odds-bot/deploy/backup.sh
rclone ls gdrive:odds-bot-backups/    # 应看到 odds-backup_日期.tar.gz
```
看到文件就说明「打包 → 上传 → 清理」整条链路 OK。

### 2.2 装定时器（systemd timer）

```bash
cd ~/odds-bot
sudo cp deploy/odds-backup.service /etc/systemd/system/
sudo cp deploy/odds-backup.timer   /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now odds-backup.timer
```

### 2.3 确认下次触发时间

```bash
systemctl list-timers odds-backup.timer
```
`NEXT` 那列应显示明天（或今天）04:00（北京时间）。配置里已写死
`Asia/Shanghai` 时区，服务器是 UTC 也能对准北京 4 点，不用你换算。

> `Persistent=true`：万一 4 点整机器在重启/关机，开机后会补跑一次错过的备份。

## 第 3 部分：新服务器一条命令恢复

场景：旧服务器崩了/换机，在**新服务器**上还原。

### 3.1 先把代码和环境拉起来

```bash
git clone https://github.com/imLahm21/odds-bot.git
cd odds-bot
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
dos2unix deploy/*.sh 2>/dev/null || sed -i 's/\r$//' deploy/*.sh   # 净化换行
```

### 3.2 装 rclone 并授权（同第 1 部分，remote 名仍叫 gdrive）

### 3.3 拉最新备份并恢复

```bash
bash deploy/restore.sh            # 自动拉云端最新一份并解开 .env/odds.db/report/data
# 或指定某一天：
bash deploy/restore.sh --list                 # 先看有哪些备份
bash deploy/restore.sh 2026-07-05_040001       # 还原指定时间戳那份
```
恢复脚本有护栏：若当前已有 `odds.db`，会先把它另存为 `odds.db.before-restore.*`
再覆盖，不会误删你现有数据。

### 3.4 起服务

```bash
sudo cp deploy/odds-bot.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now odds-bot
journalctl -u odds-bot -f          # 看到 "Telegram bot 启动" 就成了
```

（可选）把每日备份也在新机配上：重做第 2 部分。

## 日常查验小抄

```bash
systemctl list-timers odds-backup.timer     # 下次何时备份
journalctl -u odds-backup.service -n 30      # 最近一次备份日志
rclone ls gdrive:odds-bot-backups/           # 云端现有备份（保留最近 14 天）
sudo systemctl start odds-backup.service     # 手动立即备份一次
bash deploy/restore.sh --list                # 看云端能恢复哪些日期
```

## 常见问题

- **备份没上传**：先 `rclone lsd gdrive:` 看授权是否还在；token 过期重跑 `rclone config` 的授权步骤。
- **timer 不触发**：`systemctl status odds-backup.timer` 看是否 enabled；`list-timers` 看 NEXT。
- **想改保留天数**：编辑 `/etc/systemd/system/odds-backup.service`，取消 `Environment=RETAIN_DAYS=14` 那行注释并改数字，然后 `sudo systemctl daemon-reload`。
- **磁盘紧张**：备份本地也留 14 天在 `~/odds-backups/`，可改 `LOCAL_KEEP_DIR` 或调低保留天数。
