# 凯利公式投注与仓位管理方法论

> 本文档负责「看完盘之后：买不买、买哪个、买多少」，与盘口研判规则（亚盘/大小球/凯利指数）分离。
>
> **本文档分两层，读者务必分清：**
> - **第一部分 原理层（普适、不变）**：EV / 破本线 / 凯利 / 去抽水 / p 生成链 / 串关数学。
>   这些是数学恒等式与概率定律，**对任何联赛、任何样本量都成立**，是决策的硬地基。
> - **第二部分 观测层（临时、会变、仅作弱先验）**：来自本人最近 148 笔已结算战绩（截至 2026-07）

>   任何观测层结论与当场 p_市场/edge 冲突时，**以原理层为准**。校准表由 `build_calibration.py` 低频重算。

# 第一部分 原理层（普适，不随样本变）

## 第一章 核心原理：仓位由 edge 决定，不由赔率决定

### 1.1 一句话纲领

> **仓位 ∝ edge（价值），不 ∝ 赔率。**
> 「赔率越高仓位越大」是**错误命题**——同样的 edge，赔率越高凯利反而让你下越少（防波动）。
> 只有当高赔率处**真实 edge 也更大**时，才配更大仓位。决定一切的是 edge，赔率只用来算门槛。

### 1.2 三个基本量（记熟）

| 量 | 公式 | 含义 |
|----|------|------|
| **edge（价值）** | `edge = p × 赔率 − 1` | p=你诚实估的胜率。edge>0 才可投，≤0 一律 pass |
| **满仓 Kelly 比例** | `f = edge ÷ (赔率 − 1)` | 理论最优下注占本金比例 |
| **实际下注** | `注额 = 本金 × k × max(f, 0)` | k=凯利分数，默认 **1/4**（小样本+抗波动+估计有噪声） |

### 1.3 破本线（选盘时秒判）

> 赔率 O 需要真实胜率 ≥ **1/O** 才不亏。低于这条线，edge 必为负 → pass。

| 赔率 | 1.30 | 1.50 | 1.70 | 1.85 | 2.00 | 2.20 | 2.50 | 3.00 |
|------|------|------|------|------|------|------|------|------|
| **需命中** | 76.9% | 66.7% | 58.8% | 54.1% | 50.0% | 45.5% | 40.0% | 33.3% |

### 1.4 为什么用分数凯利（不用满仓）

满仓凯利在**估计误差**下极易剧烈回撤。你的 p 靠主观估计（含噪声），一旦高估 p，满仓凯利会让你下到 2 倍于最优，长期反亏。**1/4 凯利**牺牲约一点长期增速，换取生存概率与心理可执行性；对 p 更有把握（大样本、强信息差）时可放宽到 1/2。这是**对估计误差的普适对冲**，与具体样本无关。

## 第二章 p 生成链：每场先算 p，再谈仓位

> 替代「拍脑袋估一个 p」或「固定 p=0.66」。p 不是凭感觉，是「市场共识为底 + 你被验证过的判断力微调」。

**五步：**

```
步骤1  p_市场 = 多庄平均去抽水(即时节点)         ← 客观主锚，不漂移
步骤2  自评置信度（按 SOP 打分，动作不变）
步骤3  p_校准 = 查校准表(置信度→真实胜率)         ← 会漂移，仅微调（观测层，第七章）
步骤4  w = 按校准桶样本量/稳定度定权重
       p_最终 = w·p_校准 + (1−w)·p_市场
步骤5  edge = p_最终 × 赔率 − 1 → 定仓位(第三章)
```

### 2.1 步骤1：多庄平均去抽水（p_市场，取「即时」节点）

1. 取即时节点**所有有数据的大庄**，对每个盘口求各庄隐含概率 `1/赔率` 的**跨庄平均**。
2. 该盘口所有选项的平均隐含相加 = 合计（>1，多出即抽水）。
3. 每选项 `p_市场 = 平均隐含 ÷ 合计`（归一去抽水）。

> 例（胜平负 1.37/5.60/8.75）：隐含 0.730/0.179/0.114，合计 1.023，去抽水主胜 = 0.730/1.023 = **71.4%**。
> 让球/大小球两项同理（分母 = 两项隐含之和）。

**为何多庄平均而非单庄**：抗单庄挂错盘/噪声，比首选庄单庄口径更稳。缺即时节点则退临场④→③→…。

### 2.2 步骤3~4：p_校准 与权重 w

- `p_校准`：把自评置信度丢进第七章校准表，读出该区间历史真实胜率。
- `w`：**样本越多、越稳，w 越大（越敢偏离市场）**。见第七章 7.3。样本少 → w 低 → p_最终 ≈ p_市场。

### 2.3 核心纪律：edge 从哪来

p_最终 与 p_市场 的**差**，才是你的真 edge 来源。**这个差必须建立在「市场看轻的具体信息」上**（伤停/轮换/战意/赛程疲劳/长期跟踪/主客场特性），不是「我感觉主队强」——那市场早算进赔率了。

> **无具体信息差 → p_最终 ≈ p_市场 → edge ≈ 0 → pass。** 这是普适的，与你处在哪个置信度区间无关。

## 第三章 仓位定量：$100 本金的单位制

### 3.1 单位制（资金管理骨架，普适）

| 项目 | 规则 |
|------|------|
| **1 单位** | 本金的 1% = **$1**（本金 $100 时） |
| **单注区间** | 按 edge 与凯利浮动，常规 2–12 单位（$2–$12） |
| **单注上限** | ≈ 12 单位（$12），防单场过度暴露 |
| **单日总下注上限** | ≈ 15 单位（$15）；超了砍掉 edge 最低的注 |
| **动态再定基** | 本金每变动 ±20% 重算单位（跌 $80→单位 $0.8；涨 $120→单位 $1.2） |
| **止损闸** | 本金跌破 $60（−40%）停手复盘模型，不追 |

> 动态再定基是凯利复利防爆仓的核心：**赢了顺势放大，亏了自动收缩。**

### 3.2 注额公式（唯一算法，用当场 p_最终）

```
注额 = 本金 × k × edge / (赔率 − 1)
     = 100 × 0.25 × (p_最终 × 赔率 − 1) / (赔率 − 1)      （k=1/4，本金 $100）
```

- `edge ≤ 0` → 注额 ≤ 0 → **pass**。
- **赔率越高、注额未必越大**：注额由 edge 驱动，不是赔率。
- 快查参照（当 p≈0.66 时，仅示意，实战以 p_最终 代入自算）：

  | 赔率 | 1.60 | 1.80 | 2.00 | 2.20 | 2.50 | 3.00 |
  |------|------|------|------|------|------|------|
  | 注额 | $2.3 | $5.9 | $8.0 | $9.4 | $10.8 | $12.3 |

### 3.3 凯利分数 k 按证据强度调节（连续，非开关）

| 证据强度 | k | 场景 |
|---------|---|------|
| 强（大样本校准 + 明确信息差 + 多公司协同） | 1/2 | 少见，能力圈内高确信 |
| 中（校准可信 + 有信息差） | 1/4 | **默认** |
| 弱（样本少 / 信息差模糊 / edge 薄） | 1/8 | 试探性小注 |
| 无（无信息差 或 edge≤0） | 0 | pass |

> **这取代了「置信度硬闸门」**：不是「<70 一律禁投」，而是**证据越弱、凯利分数越小**，自然收敛到很小甚至 0。低置信度只要有扎实的 p_市场 edge + 具体信息差，仍可用 1/8 凯利小注参与。

## 第四章 选哪个玩法：edge 是唯一裁判

### 4.1 把每个候选换算成同一个数

一场给出胜平负/双重机会/让球/大小球/双进等多个出口。**不要问「买哪个玩法」，要问「哪个 选项+赔率 组合 edge 最高且为正」**。玩法只是表达判断的出口，edge 是统一标尺。

**流程**：
1. 对该场诚实估 P(主胜/平/客胜)（估法见 4.3）；
2. 由概率推出每个候选选项的 p（让球用胜平负概率合并，大小球单独估）；
3. 各候选算 `edge = p_最终 × 赔率 − 1`；
4. **投 edge 最高的正值项**；全为负 → **pass**。

### 4.2 市场无抽水概率（算 edge 的锚，即 p_市场）

```
主胜概率 = (1/主胜赔率) / (1/主胜 + 1/平 + 1/客胜)
```
分母减 1 = 该场抽水率。让球/大小球两项时分母 = 两项隐含之和。多庄时先跨庄平均再归一（第二章 2.1）。

### 4.3 如何估三个概率（唯一主观输入，三选一）

1. **从市场微调法（推荐）**：拿 p_市场 当起点，只问「哪一项被高估/低估」。没理由改 → 照抄 → pass。有信息差 → 只动那一项，三数和保持 100%。
2. **先定平局再分胜负**：强强对话平局偏高，先定平概率，余下按强弱分主客。
3. **进球流切入**：先判进球多不多（大小球），低比分利平局与弱队受让。

### 4.4 BCG 赔率提示（不折扣，只提醒）

p_市场 用**大庄多庄平均**算，但你**实际下注在 BCG**。BCG 抽水通常更高、赔率更差，**故算出的 edge 偏乐观**。
- 不自动打折、不要求手动输 BCG。
- **认知纪律**：对薄 edge（**< +5%**）更保守——这点优势很可能被 BCG 更高的抽水吃掉。edge 明显厚（>+10%）时才有安全垫。

### 4.5 完整示例：三玩法并排算 edge，选最高下注

> 演示「三种玩法同一套公式、各算各的 edge、选 edge 最高者下注」。假设本金 $100、k=1/4、该场自评置信度 68。

**这一场的候选**（p_市场 由即时节点多庄去抽水得到；p_最终 = w·p_校准 + (1−w)·p_市场，此处 w=0.35）：

| 玩法 | 赔率 | 去抽水口径 | 门槛=1/赔率 | p_市场 | p_最终 | **edge=p_最终×赔率−1** | 凯利 f=edge/(赔率−1) | 注额=100×¼×f |
|------|------|-----------|------------|--------|--------|----------------------|---------------------|--------------|
| 胜平负-主胜 | 1.80 | 三项(主/平/客) | 55.6% | 57% | 59% | 1.80×0.59−1 = **+6.2%** | 0.062/0.80=0.078 | **$1.9** |
| 让球-主-1 | 2.00 | 两项(主让/客受) | 50.0% | 52% | 54% | 2.00×0.54−1 = **+8.0%** | 0.080/1.00=0.080 | **$2.0** |
| 大小球-大2.5 | 1.90 | 两项(大/小) | 52.6% | 55% | 57% | 1.90×0.57−1 = **+8.3%** | 0.083/0.90=0.092 | **$2.3** |

**决策流程（严格按第四章）：**
1. 三个玩法**去抽水口径不同**（胜平负三项、让球/大小球两项），但算出 edge 后**进的是同一个凯利公式**。
2. 三个 edge 都为正，但**不是三个都下**——选 **edge 最高的「大 2.5」（+8.3%）**，下 **$2.3**。
3. 其余两项即使正 edge 也让路（集中在最高 edge；除非刻意分散且各自 edge 都厚）。
4. **若三项 edge 全为负 → 整场 pass**（空仓也是决策）。
5. **BCG 复核**：最高 edge +8.3% > +5%，安全垫尚可；若只有 +3% 这种薄 edge，考虑 pass 或降到 k=1/8。

> **要点**：没有「让球固定 $5、胜平负固定 $10」这种按玩法写死的金额。
> **每个玩法各自算 edge → 同一套破本线/凯利/上限 → 选 edge 最高者下**。玩法只是出口，edge 是唯一标尺。

## 第五章 串关：默认拆单关（数学原理）

### 5.1 为什么串关是负 edge 放大器

1. **抽水按腿连乘**：单场抽水约 5%，2 串连乘两次、3 串三次。若单腿 edge 微弱，串起来必沉水下。
2. **需全中**：单腿 65% 时，2 串 ≈ 0.65²=42%，3 串 ≈ 27%。方差急剧放大。
3. **拆单关可部分赢**：3 个独立单关允许中 2 输 1 仍净赚，且各自按 edge 独立定量。
4. **估计误差也连乘**：每腿高估 5%，2 串真实胜率打 0.90 折、3 串打 0.86 折，乐观偏差被指数放大。

### 5.2 唯一可翻正的情形：BCG 3串1 ×1.15

BCG 平台 **3串1 有 ×1.15 增益、2串1 无增益**。增益乘在总赔率上，能抵消抽水放大：

- 三腿各含约 5% 抽水，串起来抽水放大 ≈ `1 − 0.95³ = 14.3%`。
- ×1.15 ≈ 把总赔率抬高 15% > 14.3% → **数学上 3串1 可翻正**。
- **但增益是乘法，只放大已有 edge 的方向**：三腿**全部**正 edge 才可串；一腿 −EV，×1.15 也救不回。

**准入门槛（3串1，全满足）**：① 三腿**每腿** edge>0 且证据达「中」以上；② 优先低/正相关腿；③ 总注额仍受单注上限约束。

> ⚠️ **程序暂不支持多场自动分析，本条靠人工执行**：逐腿算 p_最终/edge，确认全正再串。
> **2串1 无增益 → 默认拆单关。** 拿不准 → 拆。同场**正相关**腿（判某队大胜时「该队赢 + 大球」）属相关性套利，另算。

### 7.1 怎么用这张表（关键，回应「不能太绝对」）

- 这张表给的是 **p_校准** 和 **w 的先验**，**不是「能不能下」的开关**。
- **低置信度（<70）不等于禁投**：它意味着「你的自评在这一段历史上预测力弱」，所以 **p_校准 权重 w 调低、更依赖 p_市场**，且凯利分数取小（1/8）。**只要 p_市场 给出扎实 edge + 你有具体信息差，低置信度场仍可小注参与。**
- **70–74 是历史顺手区**：w 可给高、凯利分数可到 1/4~1/2。但这也只是先验，仍需当场 edge>0。
- **74+ 谨慎**：历史显示这里易过度自信，暂按 70–74 档定量、不额外加码——但这是**观察到的偏差提醒，不是禁令**，若当场 edge 与信息差都硬，照 edge 走。

### 7.2 为什么不设硬闸门

置信度尺子在随模型更新漂移，样本又小（每桶十几到四十几笔）。用一条「<70 一律 pass」的硬线，会：① 把噪声当铁律；② 错杀低置信度里真正有 edge 的场。**正确做法是让证据强度连续地决定凯利分数（第三章 3.3）**，弱证据自然收敛到极小注或 0，而不是一刀切。

### 7.3 融合权重 w（样本量/稳定度定，越稳越敢偏离市场）

| 校准桶样本量 | w | 含义 |
|-------------|---|------|
| < 15 笔 | 0.2 | 样本太少，几乎只信 p_市场 |
| 15–30 笔 | 0.35 | 中等，谨慎微调 |
| > 30 且 ROI 稳定 | 0.5 | 敢在能力圈内偏离市场 |

> 目前唯一敢给 w=0.5 的是 70–74 桶（43 笔、ROI 稳定为正）。其余桶样本少或不稳，w 取低。

## 第八章 数据时效：p_市场 取哪个节点

- **优先取「即时」节点**（临场知情资金/伤停/天气已注入，预示力最强）。
- 缺即时 → 退「临场④(-0.5h)」→「临场③」→ 依次前推。
- 距开球越远，p_市场 越不含临场信息、edge 越不可靠——**与 CLAUDE.md 步骤7「数据时效性扣减」联动**：距开球越远，对薄 edge 越该 pass 或降凯利分数。

## 第十章 下注前 checklist（每注执行）

1. **p 生成链**：算 p_市场（即时节点多庄去抽水）→ 查校准表得 p_校准 → 按样本量定 w → p_最终。
2. **信息差**：p_最终 偏离 p_市场，是基于市场看轻的具体信息，还是脑补？脑补 → pass。
3. **选项 edge**：各候选算 `p_最终×赔率−1`，挑最高正值；全负 → pass。
4. **证据强度定凯利分数**：强 1/2 / 中 1/4 / 弱 1/8 / 无 0（第三章 3.3）。低置信度可小注，不禁投。
5. **BCG 复核**：edge < +5%？→ 更保守/pass（BCG 抽水吃薄 edge）。
6. **定量**：`注额 = 100 × k × edge/(赔率−1)`；赔率≤破本线 pass。
7. **组合上限**：单注 ≤$12、单日 ≤$15；串关仅 BCG 3串1 全腿达标才允许。
8. **记录**：记下【置信度 + 你认为的公允赔率 + 下注理由(信息差)】，供 `build_calibration.py` 事后校准。

> **附**：若每注补记一列「我认为的公允赔率」，即可算**事前 edge** 并自动校准，比置信度反推精确一个量级。
> 校准表由 `build_calibration.py` 从 `竞彩.xlsx` 低频重算（每~30笔/每周），用法见 `build_calibration.md`。
