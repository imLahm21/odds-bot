# H2H 主场碾压 + 本赛季实力接近 = H2H 权重下调

## 核心规则

**本赛季实力接近时，H2H 历史碾压优势已被抵消**

当以下条件同时满足时，H2H 权重应降至 **20%**，本赛季状态权重上调至 **30%**：
- H2H 主场不败率 ≥ 80%（历史碾压优势）
- 但本赛季积分差 ≤ 3 分（排名相邻或接近）
- 客队本赛季有大胜强队记录（如 4-1 胜前 5 名球队）

此时**不可机械套用** [[feedback_csl_home_rebound_potential]]（H2H 权重上调至 35%），因为本赛季实力已接近。

## Why

**本赛季状态 > 历史交锋**：
- 积分差 ≤ 3 分 = 实力接近，H2H 历史碾压优势已被本赛季状态抵消
- 客队本赛季大胜强队 = 状态上升，非历史弱旅
- H2H 主场平局率 ≥ 30% = 平局是高概率备选，非"主场必胜"

**中超球队状态波动大**：
- 本赛季排名反转常见（如上赛季强队本赛季降级区）
- H2H 历史碾压优势在本赛季可能完全失效

## How to apply

### 验证清单

在套用 [[feedback_csl_home_rebound_potential]] 前，必须逐项验证：

1. **本赛季积分差是否 ≤ 3 分？**
   - 是 → H2H 权重下调至 **20%**，本赛季状态权重上调至 **30%**
   - 否（积分差 > 3 分）→ H2H 权重维持 35%

2. **客队本赛季是否有大胜强队记录？**
   - 是（如 4-1 胜前 5 名）→ 状态上升，H2H 权重下调至 **20%**
   - 否 → H2H 权重维持 35%

3. **H2H 主场平局率是否 ≥ 30%？**
   - 是 → 平局是强备选，H2H 权重下调至 **20%**
   - 否 → H2H 权重维持 35%

### 权重分配

| 场景 | H2H 权重 | 本赛季状态权重 | 平局备选 |
|------|---------|--------------|---------|
| H2H 碾压 + 积分差 ≤ 3 分 | **20%** | **30%** | 强备选 |
| H2H 碾压 + 积分差 > 3 分 | **35%** | **20%** | 弱备选 |
| H2H 碾压 + 客队大胜强队 | **20%** | **30%** | 强备选 |
| H2H 碾压 + 主场平局率 ≥ 30% | **20%** | **25%** | 强备选 |

### 置信度调整

- H2H 碾压 + 积分差 ≤ 3 分 → H2H 权重 20%，置信度上限 70 分
- H2H 碾压 + 积分差 > 3 分 → H2H 权重 35%，置信度上限 75 分
- H2H 碾压 + 客队大胜强队 → H2H 权重 20%，平局置信度 +10 分

## 案例

**河南 0-2 浙江（2026-05-30 中超）**：
- H2H 主场不败率 100%（3 胜 2 平 0 负）
- 本赛季积分差 1 分（河南 15 分第 8，浙江 14 分第 9）
- 浙江本赛季 4-1 胜山东泰山（5 名）
- H2H 主场平局率 40%（5 场 2 平）
- 误判：机械套用 [[feedback_csl_home_rebound_potential]]，H2H 权重 35%
- 实际：积分差 1 分 + 浙江大胜强队 → H2H 权重应为 20%，本赛季状态权重 30%
- 浙江 0-2 客胜河南

## 边界条件

**触发 H2H 权重上调至 35% 的场景**（[[feedback_csl_home_rebound_potential]]）：
1. H2H 主场不败率 ≥ 60% + 积分差 **≥ 4 分**（主队领先）
2. H2H 主场不败率 ≥ 80% + 积分差 **≥ 8 分**（主队领先）→ 权重上调至 35%（[[feedback_h2h_home_dominance_same_direction]]）
3. 客队本赛季**无**大胜强队记录
4. H2H 主场平局率 < 30%
