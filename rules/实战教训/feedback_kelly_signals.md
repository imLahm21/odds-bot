# 凯利信号研判

> **触发条件**：任一公司凯利出现极端值（>1.05 / <0.95）、方向反转、或与盘口信号矛盾；初盘开小（浅盘）场景读凯利前；即时节点凯利跳变。

## A. 即时节点凯利异动：趋势延续 vs 突变反转

即时节点（封盘前最后一次调赔）的凯利突变需区分两种情况：

| 情况 | 特征 | 正确解读 |
|------|------|----------|
| 趋势延续型 | 凯利变化方向与前 3~4 个节点一致，幅度递增 | 真实信号，可信 |
| 突变反转型 | 方向与前序节点相反，或**单节点变化 > 前序 2~3 节点累计变化** | 庄家封盘前诱导操作，**反向解读或忽略** |

例：临场①→③主凯利缓升 +0.012，临场③→即时单节点 +0.026（超前序总和）+ 平局赔率跳升 → 突变反转型，反向解读为诱下。

## B. Pinnacle 全面偏移时凯利间距 < 0.05 必须忽略

Pinnacle 欧赔三项凯利全程同侧（如全 > 1.0，全面偏移庄家）且各项间距 < 0.05 时，**严格忽略该公司凯利差异**，不可作为主要依据（如间距 0.023/0.027 被当作"平局易打出"依据 → 错误强化平局信号）。

## C. 浅盘场景下凯利信号的结构性失真（灰色地带）

> 来源：浙江 4-1 山东泰山（初盘开小 + 客胜凯利 1.042 误判为真实信号）。

初盘开小时，客胜凯利 > 1.0 可能是**盘口浅开的数学结果**（客队让球负担轻 → 客胜赔率结构性偏高），非庄家真实看好客队。阈值放宽：

| 初盘性质 | 客胜看好 | 客胜不看好 | 灰色地带 |
|---------|---------|-----------|---------|
| 合适/开大 | < 0.96 | > 1.05 | 0.96~1.05 |
| **开小** | **< 0.95** | **> 1.08** | **0.95~1.08** |

镜像规则（来源：Chelsea 2-1 Tottenham）——浅盘（≤ -0.25）场景下**亚盘主凯利**：

| 初盘性质 | 看好上盘强信号 | 灰色地带 |
|---------|------------------|---------|
| 合适/开大 | < 0.90 | 0.90~0.95 |
| **开小或浅盘** | **< 0.85** | **0.85~0.95** |

灰色地带凯利**不可单独定性**，必须验证：升盘后主水方向（反升 ≥15 点 = 失效）、即时欧赔延续性（回调 ≥0.05 = 失效）、升盘后主凯利（>1.08 = 失效）。「升盘前凯利趋势优先」（[feedback_sync_pricing.md](feedback_sync_pricing.md) A2）在「初盘开小 + 凯利落于灰区」时**不适用**。

## D. 优先级反转：Pinnacle 客胜凯利单边极值

**D1：客胜凯利 < 0.96 被掩盖的真实信号**（来源：云南玉昆 1-2 浙江）：

| 信号组合 | 解读 |
|---------|------|
| 主胜凯利 0.99~1.0（多公司压制）+ 客胜凯利 > 1.0（多公司） | 主胜被压制，可信 |
| **主胜凯利 0.99~1.0（多公司压制）+ Pinnacle 客胜凯利 < 0.96** | **优先级反转：主胜压制视为诱导，客胜低凯利视为真实** |
| 主胜凯利 < 0.96（多公司） + 客胜凯利 > 1.0（多公司） | 主胜真实看好 |

Pinnacle 任一方向凯利 < 0.96 必须列入核心信号清单（不可仅列辅助）。

**D2：客胜凯利 > 1.05 的镜像**（来源：上海海港 2-2 浙江）：**Pinnacle 客胜凯利 > 1.05 + 多家协同压主胜**并存时，不可解读为"客胜被抛售"——实质是庄家在客胜方向开放高赔付敞口、**不惧客胜打出**，反向支持客胜。Pinnacle 任一方向凯利 > 1.05 时必须列入核心信号清单。

## E. 凯利反转优先于绝对值（方向 > 数值）

> 来源：河南 0-2 浙江（2026-05-30 中超）。前 8 节点主凯利 0.853~0.887，临场④主凯利 +0.073 / 客凯利 -0.061，仍以"绝对值 <0.96"维持原判 → 错误。

前期凯利持续某方向（如主凯利 < 0.90 持续 3+ 节点），临场④出现主凯利反向变化 ≥ 0.05 **且**客凯利同步反向 ≥ 0.05 = **凯利体制反转**，即使绝对值仍 < 0.96 也判反转：

| 凯利变化幅度 | 判定 |
|------------|------|
| **≥ 0.05** | 反转信号 → 庄家态度反转 |
| 0.03~0.05 | 灰色地带 → 结合盘口/欧赔判断 |
| < 0.03 | 微调 → 维持原判断 |

主凯 ≥ +0.05 且 客凯 ≤ -0.05 → 看好客队；反之看好主队。凯利是庄家对打出概率的动态定价，临场④是最后修正窗口，最新态度 > 历史态度。

## F. Pinnacle 亚盘双重极端信号 = 强制反转或放弃

> 来源：重庆铜梁龙 2-3 北京国安（2026-05-30 中超）。双重红灯仅作"扣分"处理仍推主队 → 错误。存档：[20260531_case_03_chongqing_vs_beijing.md](20260531_case_03_chongqing_vs_beijing.md)。

| Pinnacle 亚盘凯利信号 | 强度 | 处理 |
|-------------------|------|------|
| 主凯 >1.05 或 客凯 <0.95（单项偏离） | 中 | 扣 5~10 分，该方向列次选 |
| 主凯 >1.08 或 客凯 <0.90（单项极端） | 强 | 扣 10~15 分，该方向列次选 |
| **主凯 >1.08 且 客凯 <0.90（双重极端，临场①~③）** | **极强** | **强制反转结论或放弃该场** |

Pinnacle 不限注、赔率最接近真实概率，双重极端 = 主客两维度同时确认（主凯 >1.08 = 不惧主队打出；客凯 <0.90 = 真实看好客队）。方向明确，应反转而非降置信度。

**临场④凯利回归正常 ≠ 信号消解**：盘口调整（如 +0.5→+0.25）导致主水下降、主凯利自然回落，是数学结果。临场③已触发双重极端则维持判定。唯一例外：临场④盘口反向调整且凯利完全反转（主凯 <0.95、客凯 >1.05）。

**边界（关闭强制反转）**：极端动力差异场景（保级死战 vs 无欲无求，满足 ≥3 项）→ [feedback_csl_fundamentals.md](feedback_csl_fundamentals.md) E 节。

## G. 平局凯利信号

- **任一公司平局凯利 < 0.96 = B/C 冲突**（来源：霍芬海姆 3-3 斯图加特，Tipico 平局凯利 0.935 被定性"基本一致" → 错失平局）。算法 B 推某方胜出 + 平局凯利 < 0.96 报警：不可定性"基本一致"，置信度降 10~15 分，平局至少列次选；与 H2H 平局率 ≥ 60% 叠加时平局升首选（[feedback_h2h_weight.md](feedback_h2h_weight.md) A 节）。
- **平局凯利 > 1.03 + 平局赔率持续上升 > 30 点 = 平局首选**（来源：Bournemouth 1-1 Man City，平赔 4.11→4.49 被忽略）。平局凯利 > 1.03 = 庄家压制平局赔付，看好平局打出。
- **客凯利压制 + 平局凯利 > 1.0 = 看好平局而非客胜**（来源：同上）。凯利压制 = 降低赔付，不等于压制打出概率，方向须结合平局项判定。

## H. 非核心庄单项平局凯利报警：双锚确认门槛

> 来源：KA阿克雷里 3-2 IA阿克拉内斯（2026-07-13 冰岛超）。原判错误地将威廉希尔平局凯利0.949叠加其他低平赔信号，把平局升为首选。存档：20260713_case_11_KA_Akureyri_vs_IA_Akranes.md。

**规则 H1：单项报警只判冲突。** 任一非核心公司平局凯利 < 0.96，先标记B/C冲突并降低平局确定性，不得凭单点信号直接定向平局。

**规则 H2：双锚不确认不得升首选。** 若Bet365、Pinnacle均无平局方向确认，非核心庄报警只可将平局列为次选或风险，不得独立将其升为首选。

**规则 H3：低平凯利不能区分平局与冷方胜。** 与热门方浅盘不升盘并存时，该信号至多强化“热门不宜追”；应再用主场、近况与历史交锋区分“平局”还是“冷方胜”。

**与 G 的区别**：G 节规定任一公司平局凯利 < 0.96 即触发B/C冲突并将平局列入次选；H 节补充非核心庄单点报警的升级门槛——没有核心双锚确认时不得升为首选，也不得把“热门不出”自动收敛为平局。

## 本主题检查项

- [ ] 即时节点凯利异动是趋势延续型还是突变反转型？（单节点变化 > 前序累计 = 反向解读）
- [ ] Pinnacle 三项凯利全面偏移（同侧）且间距 < 0.05？→ 忽略该公司凯利信号
- [ ] 初盘开小？→ 客胜凯利 1.03~1.05 / 亚盘主凯利 0.85~0.95 均为灰色地带，不可单独定性
- [ ] Pinnacle 客胜凯利 < 0.96？→ 列核心信号；与"多家轻压主胜"并存触发优先级反转
- [ ] Pinnacle 客胜凯利 > 1.05？→ 与"协同压主胜"并存触发反向优先级反转
- [ ] 临场④凯利反转（主凯 ≥+0.05 且客凯 ≤-0.05）？→ 方向优先于绝对值
- [ ] Pinnacle 亚盘双重极端（主凯 >1.08 且客凯 <0.90，临场①~③）？→ 强制反转或放弃；临场④回归正常不消解；先排除极端动力差异
- [ ] 任一公司平局凯利 < 0.96？→ 判 B/C 冲突（非"基本一致"），平局至少次选
- [ ] 平局凯利 > 1.03 或平局赔率升 > 30 点？→ 平局纳入首选评估
- [ ] 非核心庄单项平局凯利 < 0.96？→ 标记B/C冲突并降低平局确定性，不得凭单点定向
- [ ] 平局报警但Bet365、Pinnacle均无确认？→ 平局只列次选或风险，不得升为首选
- [ ] 低平凯利与热门方浅盘不升盘并存？→ 先否定热门，再以主场、近况和历史交锋区分平局或冷方胜
